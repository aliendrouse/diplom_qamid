# План тестирования и автоматизации приложения «Мобильный хоспис»

## 1. Объект тестирования
Мобильное приложение «Мобильный хоспис» (версия 1.0.0) на Android.

## 2. Цель тестирования
Проверка функциональности, удобства использования и стабильности приложения для работы с новостями хосписа.

## 3. Границы тестирования (Scope)
* **Будем тестировать (In Scope):** Проверяем форму входа, работу всех кнопок навигации, формы создания и редактирования новостей (как они принимают текст, даты, эмодзи). Смотрим, как работают фильтры и сортировка. Проверяем удаление карточек новостей и системные всплывающие окна.
* **Не будем тестировать (Out of Scope):** Регистрацию новых пользователей и сброс пароля. Нагрузочное тестирование сервера мы тоже не делаем — проверяем работу одного клиента на эмуляторе.

## 5. Типы тестирования
* **Функциональное тестирование:** Позитивные и негативные сценарии, граничные значения.

* **UI/UX тестирование:** Визуальное оформление и адаптивность.

* **Тестирование навигации:** Переходы вперед и возврат назад.

* **Тестирование обработки ошибок:** Валидация полей ввода, сообщения об ошибках.

## 4. Данные для входа
- Логин: login2
- Пароль: password2

## 5. Где тестировали
- Эмулятор Android API 29 (Pixel 4).

## 6. Баги, которые нашли
1. **Новости не создаются** (критичный баг) - при сохранении новость не появляется в списке.
2. **Дата создания показывает 58454 год** - вместо 2026.

## 7. План по автоматизации 
- **Фреймворк:** Espresso (встроен в Android Studio).
- **Язык:** Java/Kotlin.
- **Сборка:** Gradle.
- **Отчеты:** Allure (наглядные отчеты о прохождении тестов).

### Планируемые к автоматизации сценарии:
1. Вход в приложение (успешный и с ошибками).
2. Фильтрацию новостей.
3. Редактирование новостей.
4. Удаление новостей.

### Сценарии, не подлежащие автоматизации:
- Создание новостей (баг, который мы нашли, пока не починят).
- Просмотр цитат и раздела "О приложении" (там только статичная информация, теряется смысл).
